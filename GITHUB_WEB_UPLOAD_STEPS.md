# 📤 GitHub 网页端上传代码 - 详细步骤

## 🎯 步骤 1：打开你的 GitHub 仓库

1. 在浏览器中访问：
   ```
   https://github.com/LindaD0710/ielts-novel-flow
   ```

2. 你应该会看到一个空的仓库页面

---

## 🎯 步骤 2：进入上传页面

**如果你的仓库是空的**，页面中间会显示：

- "Quick setup" 区域
- 或者 "No files in the repository" 提示

找到并点击：**"uploading an existing file"** 链接（蓝色链接）

**如果没有看到这个链接**：

1. 点击页面右上角的 **"Add file"** 按钮（绿色按钮）
2. 在下拉菜单中选择 **"Upload files"**

---

## 🎯 步骤 3：上传文件

### 方法 A：拖拽上传（推荐）

1. 打开 Finder（文件管理器）
2. 导航到项目文件夹：
   ```
   /Users/lindadong/Desktop/小创意/【破局】AI编程出海/IELTS Novel Flow
   ```
3. **选择以下文件和文件夹**（按住 Command 键多选）：
   - `package.json`
   - `package-lock.json`
   - `next.config.js`
   - `tsconfig.json`
   - `tailwind.config.ts`
   - `postcss.config.js`
   - `.gitignore`
   - `app` 文件夹（整个文件夹）
   - `components` 文件夹（整个文件夹）
   - `src` 文件夹（整个文件夹）
   - `types` 文件夹（整个文件夹）
   - `data` 文件夹（整个文件夹）
   - `postcss.config.js`（如果还没有选择）

4. **不要选择**：
   - ❌ `node_modules` 文件夹
   - ❌ `.next` 文件夹（如果有）
   - ❌ `.git` 文件夹
   - ❌ `tools` 文件夹（可选，部署不需要）

5. 将选中的文件和文件夹**拖拽**到 GitHub 的上传区域

### 方法 B：点击选择文件

1. 点击上传区域的 "choose your files" 链接
2. 在文件选择器中选择文件（可以按住 Command 键多选）
3. 一次选择所有需要的文件和文件夹

---

## 🎯 步骤 4：等待上传完成

1. 上传过程中会显示进度条
2. 文件较多（约 60-70 个文件），可能需要几分钟
3. 上传完成后，文件名会显示在上传区域

---

## 🎯 步骤 5：提交文件

1. 滚动到页面底部
2. 找到 "Commit changes" 区域
3. 填写 Commit message（提交信息）：
   ```
   Initial commit: 32 novels ready for deployment
   ```
4. 确保选择 "Commit directly to the main branch"
5. 点击 **"Commit changes"** 按钮（绿色按钮）

---

## ⏰ 步骤 6：等待上传完成

1. 提交后，页面会显示上传进度
2. 上传完成后，会跳转到仓库页面
3. 你应该能看到所有上传的文件和文件夹

---

## ✅ 步骤 7：回到 Vercel

1. 代码上传成功后，回到 Vercel 项目页面：
   ```
   https://vercel.com/dashboard
   ```
2. 进入 `ielts-novel-flow` 项目
3. 等待几秒钟，Vercel 会自动检测到新的代码
4. 自动开始部署
5. 可以在 "Deployments" 标签页查看部署进度

---

## 💡 提示

- **如果上传中断**：可以重新上传，GitHub 会合并文件
- **如果文件太多**：可以分批次上传（先上传核心文件，再上传其他）
- **上传速度**：取决于你的网络速度，耐心等待即可

---

## 🎯 开始上传

现在按照步骤操作：

1. 访问 https://github.com/LindaD0710/ielts-novel-flow
2. 点击 "uploading an existing file" 或 "Add file" → "Upload files"
3. 选择并上传文件
4. 提交更改

**上传过程中如果遇到问题，随时告诉我！**
